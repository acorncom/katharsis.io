Request Parameters

In contrast to JSON:API standard due to server request string length limitation katharsis consumes request params as JSON Objects and Arrays.

Katharsis provides a set of parameters which change the returning JSON object.

Sorting

Sorting information for the repositories can be achieved by providing soring information in sort parameter. The structure of the parameter should be as follows:

  1. The value passed should be a valid JSON object
  2. The keys should be paths of the attributes of a resource
  3. The key's value should contain one of the following strings: asc or desc

An example of the soring parameter: {"name": "asc"}.

Inclusion of Related Resources

All of the resources, for which a relationship has been defined, can be returned along with the main resource. The parameter include should contain those requests and its structure should be as follows:

  1. The value passed should be a valid JSON array
  2. The values of the array should be paths of the attributes of a resource

Examples of the inclusion parameter: ["project"], ["project.author", "team"].

Sparse Fieldsets

Requested resource from endpoint can be limited to specific fields in the response. In order to provide set of included fields parameter fields should should be as follows:

  1. The value passed should be a valid JSON array
  2. The values of the array should be list of the attributes of a resource
  3. Attributes id and type will be always returned regardless of containment of sparse fieldset

Examples of the fields parameter: ["name"], ["description", "project.author"].
